What is value.json?

A value chain refers to the full lifecycle of a product or process, including material sourcing, production, consumption and disposal/recycling processes.

value.json is a standard for tracking and attributing value across software projects, making the often invisible contributions and inspirations in our development ecosystem visible and quantifiable.

Key Questions for Every Implementation

  • • What is the added value compared to pre-existing systems?
  • • What inspirations led to the creation of this new system?
  • • Who/what does the author attribute the most value to?
  • • Which inspirations are obvious and therefore not attributed explicitly?

The Schema

type Value = {
  allocation: {
    /**
     * A URL linking to the origin of the value. Should be as specific as possible.
     *
     * Examples:
     * - https://github.com/janwilmake (myself)
     * - https://openapis.org (an organization)
     * - https://claude.ai (a tool)
     */
    url: string;
    
    /**
     * Numeric relative representation of attributed value
     */
    value: number;
    
    /**
     * Why this value is attributed to this source
     */
    description: string;
  }[];
};

If every system attributes its sources, we can calculate optimal value distribution by tracing attributions throughout the value chain.

How to Implement

To participate with the value.json convention, create a JSON file called value.json at the root of your repository or website: 

{
  "$schema": "https://valuejson.com/value.schema.json",
  "sources": []
}
